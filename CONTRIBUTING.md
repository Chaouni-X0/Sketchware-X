# دليل المساهمة في Sketchware X

نرحب بمساهماتك في مشروع Sketchware X! لمساعدتك على البدء، يرجى قراءة هذا الدليل.

## كيف تساهم

1.  **Fork** المستودع.
2.  **Clone** المستودع المتفرع إلى جهازك المحلي.
3.  أنشئ **فرعًا جديدًا** (branch) للميزة أو الإصلاح الذي تعمل عليه.
    ```bash
    git checkout -b feature/your-feature-name
    ```
4.  قم بإجراء تغييراتك.
5.  **اختبر** تغييراتك جيدًا.
6.  **Commit** تغييراتك برسالة واضحة وموجزة.
    ```bash
    git commit -m "feat: Add new feature X" # أو "fix: Fix bug Y"
    ```
7.  **Push** الفرع إلى المستودع المتفرع الخاص بك.
    ```bash
    git push origin feature/your-feature-name
    ```
8.  افتح **طلب سحب (Pull Request)** إلى الفرع الرئيسي (master) للمستودع الأصلي.

## إرشادات طلب السحب (Pull Request)

*   تأكد من أن الكود الخاص بك يتبع معايير جودة الكود الموجودة.
*   اكتب وصفًا واضحًا ومفصلاً لطلب السحب الخاص بك.
*   أضف لقطات شاشة أو مقاطع فيديو إذا كانت التغييرات مرئية.
*   تأكد من أن جميع الاختبارات تمر بنجاح.

## الإبلاغ عن الأخطاء

إذا وجدت خطأ، يرجى الإبلاغ عنه في قسم [Issues](https://github.com/Chaouni-X0/Sketchware-X/issues) في المستودع. يرجى تضمين أكبر قدر ممكن من التفاصيل:

*   خطوات إعادة إنتاج الخطأ.
*   السلوك المتوقع.
*   السلوك الفعلي.
*   إصدار Sketchware X الذي تستخدمه.
*   إصدار Android الذي تستخدمه.

شكراً لمساهمتك في Sketchware X!
